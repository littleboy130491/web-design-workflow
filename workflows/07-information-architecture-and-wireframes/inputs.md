# Inputs

- Required: `outputs/03-content-direction/site-map.md`
  - Source: previous workflow step `03-content-direction`
  - Use: page inventory and page purpose for page blueprint planning

- Required: relevant page copy from `outputs/05-copywriting/`
  - Source: previous workflow step `05-copywriting`
  - Use: realistic content length and section intent for wireframe decisions

- Required: `outputs/06-visual-direction/recommended-direction.md`
  - Source: previous workflow step `06-visual-direction`
  - Use: approved visual direction that should inform structural emphasis and layout tendencies

- Required: `outputs/06-visual-direction/visual-guidelines.md`
  - Source: previous workflow step `06-visual-direction`
  - Use: approved typography, spacing, composition, layout, and component rules that the wireframes must preserve

- Required: `outputs/06-visual-direction/style-review.md`
  - Source: previous workflow step `06-visual-direction`
  - Use: approved user feedback on style variables such as heading scale, spacing density, image prominence, border treatment, radius, shadow, and icon tone that the wireframes must preserve structurally

- Optional: `outputs/06-visual-direction/asset-sources.md`
  - Source: previous workflow step `06-visual-direction`
  - Use: approved photo and icon sources if wireframes include sourced image references or icon references
  - The executor may proceed without this file if wireframes only use labeled placeholders instead of specific sourced assets
